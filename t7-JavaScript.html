JavaScript Documentation vs. Ruby Documentation

3/1/2015

Ruby Docs are probably one of the things that make me relatively anxious in a given week. My brain just coils into an indecipherable mass when I Google something Ruby related and it seems like Stackoverflow and all other layman's terms references don't produce an answer. It is also a little disorienting that it is and isn't English. We see the articles, verbs, and prepositions of English but the nouns and verbs are just a different language entirely.

I have found JavaScript documentation to be much more legible -- though not always as applicable and friendly to error driven development as Ruby. However, the age of JavaScript means there is tons more to choose from when you get stumped and I have relished this fact as I continue to learn.
