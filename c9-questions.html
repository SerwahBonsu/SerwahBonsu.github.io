In code -- unfortunately -- there is such a thing as asking a stupid question

3/16/2015

So I really think this piece, "Writing the Perfect Question" nails it -- in that it sums up exactly what I need to be keeping in mind when I ask things online or perhaps in-person. I have been shut down on StackOverflow and Quora a time or two and it wasn't until I started to learn how to code that I learned why. Unfortunately in the world of code -- and particularly stack overflow -- there are such things as stupid questions. Below are some bullet points on avoiding getting permanently banned on the might Stack Overflow:

  • Context Matters: Always say the programming language you are referencing.
  • Headlines Matter: The abbreviated question is just as important as the unabridged version.
  • Get to the Heart of One Question: Sometimes people will ask multiple questions; one good question is ideal.
  • Sample Code and Data This is key to getting an applicable response and showing you have at least taken a whack at what you want others to help solve. Heaven helps those...

In the end, it has been informative to get to the root of how I pose questions, as I suspect it has everything to do with the answers I need.
